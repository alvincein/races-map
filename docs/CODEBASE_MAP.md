## Folder purposes

### `src/app/`
Next.js App Router. Only one route (`/`). `page.tsx` is a Server Component that fetches all displayable races and passes them as a prop into the client tree. `layout.tsx` mounts Inter and disables user-zoom on mobile. `globals.css` is the design-system stylesheet imported once in `layout.tsx`.

### `src/components/`
All UI lives here. There's no co-location of styles per-feature beyond filename pairing (`Foo.tsx` + `Foo.css`); CSS files are imported as side-effect imports from the matching TSX file. None of the components have unit tests. None use React Context.

### `src/data/`
Static, build-time data shipped with the bundle. Currently only `raceRoutes.json`. Treat this directory as "compiled assets" — anything here is regenerated, not hand-edited.

### `src/lib/`
Single-purpose utilities. Currently just the Supabase client singleton.

### `src/types/`
TypeScript type declarations. `database.ts` mirrors the Supabase schema; `filters.ts` is currently empty (see Open Questions).

### `gpx/`
Raw source files. Adding a new race route means dropping a `.gpx` (or `.xml`) here and updating `scripts/process_gpx.py`. Files are not pulled into the JS bundle — they only feed the Python pipeline.

### `scripts/`
Python helpers. `process_gpx.py` is the only one that produces a runtime artifact (`src/data/raceRoutes.json`). The rest are Supabase REST clients used to discover sub-race UUIDs for the `ID_MAPPING` dict — they're disposable spelunking tools rather than sustained infra.

### `public/`
Standard Next.js public assets. None of the SVGs here are referenced from `src/`. Safe to clean up.

### `docs/`
Documentation (these files).

## Key files and what they do

### Entry point: [src/app/page.tsx](src/app/page.tsx)
- Server Component, default export `Home`.
- Sets `export const revalidate = 3600` for hourly ISR.
- Issues `supabase.from('races').select('*, sub_races!inner(id)').not('location_lat', 'is', null).not('location_lng', 'is', null).limit(1000)`.
- Deduplicates by `id` (the `!inner` join can multiply rows).
- Has a hardcoded fallback race for when Supabase is unconfigured.
- Renders `<HomeClient initialRaces={races} />`.

### Orchestrator: [src/components/HomeClient.tsx](src/components/HomeClient.tsx)
- Defines `FilterState` (exported and re-imported by `Sidebar.tsx` and `FilterWidget.tsx`).
- Owns the entire mutable client state for the page.
- Memoizes `filteredByControls` — the result of applying the filter UI to `initialRaces`.
- `handleRaceSelect` triggers the on-demand `sub_races` fetch.
- Renders `<MapClient/>`, `<Sidebar/>`, `<FilterWidget/>`, conditionally `<ElevationWidget/>`, plus the mesh-blob background and the mobile "show list" button.

### Map and clustering: [src/components/MapClient.tsx](src/components/MapClient.tsx)
- Wraps `react-map-gl/maplibre` `<Map>`.
- `MAP_STYLES` array exports four basemap options (Carto dark/light/voyager + Esri satellite raster).
- Two memoized inner components: `RaceMarker` (single pin with hover label) and `ClusterMarker` (count circle with hover-only leaves popup).
- Uses `useSupercluster({ points, bounds, zoom, options: { radius: 50, maxZoom: 20 } })`.
- Implements custom **spiderfy** when expansion zoom exceeds 18 — fans cluster contents around the center point.
- Renders selected race's sub-race routes as GeoJSON `<Source>` + `<Layer>` pairs, with a glow halo on the focused one.
- Maintains current map bounds; re-publishes the in-bounds race subset to `HomeClient` after a 400ms debounce.
- Holds the bottom-right NavigationControl and the top-right Layers/Reset menu.

### Sidebar: [src/components/Sidebar.tsx](src/components/Sidebar.tsx)
- A single component with two render branches selected by `selectedRace` truthiness.
- **List view:** search box, sorted-by-date list of `RaceCard`s.
- **Detail view:** hero, description (with read-more), sub-races grid with stat boxes, "Επίσημη Ιστοσελίδα" link.
- On mobile, becomes a draggable bottom sheet with manually wired touch handlers.
- Imports `raceRoutes.json` to know which sub-races have a route badge.
- Note: imports `ElevationProfile` from a sibling, but does not currently render it (legacy import — see GOTCHAS).

### Filter UI: [src/components/FilterWidget.tsx](src/components/FilterWidget.tsx)
- Stateless apart from `isOpen` (the dropdown toggle).
- Reads `filters: FilterState` and calls `onChange` for every toggle.
- Hardcoded month names in Greek and distance-bucket labels.

### Elevation chart: [src/components/ElevationProfile.tsx](src/components/ElevationProfile.tsx) + [ElevationWidget.tsx](src/components/ElevationWidget.tsx)
- `ElevationProfile`: pure SVG chart (`viewBox="0 0 1000 150"`). Computes `fillPath` and `strokePath` from `data: Point[]`. Mouse + touch handlers find the closest point by horizontal proportion and call `onHover`.
- `ElevationWidget`: chrome around the chart — title, stats grid, hover indicator pill, close button.

### Supabase client: [src/lib/supabase.ts](src/lib/supabase.ts)
- 8 lines. Exports a singleton `supabase = createClient<Database>(URL, ANON_KEY)`.
- Reads `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY` from env, defaults to empty string if missing.

### Types: [src/types/database.ts](src/types/database.ts)
- Standard Supabase-style nested type. Helpers: `Race = Database['public']['Tables']['races']['Row']`, `SubRace` similarly.

### Static route data: [src/data/raceRoutes.json](src/data/raceRoutes.json)
- 10 entries currently (Athens Marathon, TeRA 60km, Zagori 5/10/21/33km, Olympus Marathon/Ultra, Metsovo Ursa 21km, Ursa 40km).
- Generated by `scripts/process_gpx.py`. Do not hand-edit.

### Build step: [scripts/process_gpx.py](scripts/process_gpx.py)
- Hardcoded paths: `GPX_DIR = '/Users/theo/Documents/Projects/races-map/gpx'`, `OUTPUT_FILE = '/Users/theo/Documents/Projects/races-map/src/data/raceRoutes.json'`.
- `ID_MAPPING` dict at the bottom: filename → sub_race UUID. **Hand-maintained.**
- Parses `.gpx` (and `.xml` Athens Marathon) via `xml.etree`, falling back through `trkpt` → `rtept` → `wpt` element lists.
- Computes haversine distance, gain, loss, max/min elevation.
- Downsamples to ~200 profile points for the chart.
- Emits `{stats, profile, geojson}` per sub-race.

## Entry points and how the app starts

### Local development
1. Install dependencies: `npm install`.
2. Provide Supabase credentials in `.env.local` (or rely on the hardcoded fallback race):
   ```
   NEXT_PUBLIC_SUPABASE_URL=https://<project>.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=<anon-key>
   ```
3. Run `npm run dev` → Next dev server on `http://localhost:3000`.
4. The single route `/` renders the map.

### Production build
1. `npm run build` → Next build (also type-checks; `eslint` not run automatically by `build` per `package.json`).
2. `npm start` → production server.

### Adding a new race route (the manual data step)
1. Drop a GPX/XML file into `gpx/`.
2. Find the matching `sub_races.id` in Supabase (use one of `scripts/find_*.py` or `list_races.py` as templates — note: each script has a hardcoded URL+publishable key).
3. Add an entry to `ID_MAPPING` in `scripts/process_gpx.py` (filename → uuid).
4. Run `python3 scripts/process_gpx.py`.
5. Commit the regenerated `src/data/raceRoutes.json` and the new GPX file.
6. The sub-race will now show a "Διαδρομή" badge and be clickable in the detail view.

### Lint and type-check
- `npm run lint` — ESLint flat config, Next core-web-vitals + TypeScript rules.
- TypeScript checking happens via `next dev` / `next build`. There is no separate `npm run typecheck` script.
